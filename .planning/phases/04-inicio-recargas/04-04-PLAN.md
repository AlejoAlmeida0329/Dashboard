<objective>
Extend the existing presenter-mode CSS contract with a cliente-foco gate. Today: `[data-presenter="on"] [data-presenter-hide]` hides Comisión, Take rate, etc. After this plan: `[data-presenter="on"][data-empresa-filter="active"] [data-presenter-empresa-hide]` ALSO hides — but only when BOTH presenter is on AND a specific empresa is selected (cliente-foco). This enables Plans 04-05/04-07/04-08 to mark elements (e.g. the entire HechosCurados container, the EmpresasActivasChart card) with `data-presenter-empresa-hide` so they auto-disappear in cliente-foco views without any JS conditional.

Purpose: CONTEXT.md cliente-foco contract requires that the 3 hechos curados hide ONLY in cliente-foco (not in plain presenter mode), and that the "Empresas activas en el tiempo" chart (which degenerates to a flat line at y=1 when empresa is filtered) hide too. Doing this in CSS preserves the project's invariant that all visibility flips through data-attributes, not React conditionals (Phase 1 decision, see STATE.md decisions list).

Output: Two file modifications — one attribute added to PresenterFrame's wrapper div, one CSS rule appended to globals.css.
</objective>

**Component body — pipeline** (mirroring bonos/page.tsx:75-144):

1. `const params = await searchParams;` then `const filters = parseFilters(params);`

2. `try { const result = await getCachedPayouts(); } catch (err) { return <Card>… error message…</Card>; }` — mirror bonos error fallback verbatim, replace "transacciones" copy with "payouts" in the error sentence.

3. `const periodOnly = filterPayoutsByPeriodOnly(result.rows, filters);` — universe filtered by date+empresa but NOT state, used for success rate denominator.

4. `const completed = filterPayouts(result.rows, filters);` — universe filtered by ALL criteria including state, used for everything else. (filterPayouts already calls filterPayoutsByPeriodOnly internally if structured that way; otherwise compute both.)

5. **Empty-state guard**: `if (completed.length === 0) { return <div className="space-y-6"><PayoutsKPICards summary={summarizePayouts([])} successRate={aggregateSuccessRate(periodOnly)} /><Card>Sin payouts en el período seleccionado</Card></div>; }` — note: success rate IS computed even on empty completed, because periodOnly may have rejected payouts (e.g. "100 attempts, 0 completed → 0% success rate" is meaningful info, not an empty state per se). Document this inline.

6. **Aggregations**: 
   ```tsx
   const summary = summarizePayouts(completed);
   const successRate = aggregateSuccessRate(periodOnly);
   const histogram = aggregateLatencyHistogram(completed);
   const destination = aggregateByDestination(completed);
   ```

7. **Render** (responsive layout):
   ```tsx
   return (
     <div className="space-y-6">
       <PayoutsKPICards summary={summary} successRate={successRate} />

       <Card>
         <CardHeader>
           <CardTitle>Latencia de payouts</CardTitle>
         </CardHeader>
         <CardContent>
           <LatencyHistogram buckets={histogram} />
         </CardContent>
       </Card>

       <DestinationSplit stats={destination} />
     </div>
   );
   ```

   Note: `LatencyHistogram` itself wraps in a Card per Plan 03-03; the outer `<Card>` here is redundant. PICK ONE — either drop the outer wrapper OR have `LatencyHistogram` not wrap. Decision: have `LatencyHistogram` NOT wrap (it returns just the chart + ResponsiveContainer); the page provides the Card chrome. This matches `BonosChart` which is just the chart (page provides Card around it). Update Plan 03-03's component IF needed during execution; document any deviation.

8. NO `verifySession()` call — the `(protected)/layout.tsx` already guards the subtree (mirrors bonos page contract).

9. NO empresa registry fetch — `DashboardHeader` already does this via `getCachedTransactions()`. The Payouts page does NOT need to read transactions; it only reads BD_Payouts. (Future: if join is required per Plan 03-01 verdict, the page may need both — in which case adding a `getCachedTransactions()` call here is one line, and React `cache()` ensures no double-fetch with DashboardHeader. Document the join decision based on Plan 03-01 outcome.)

10. **Filter propagation invariant**: write a JSDoc comment at the top stating "All four leaf renderings derive from `completed` (or `periodOnly` for successRate). Do NOT compute aggregations off `result.rows` directly — that would bypass the date/empresa filter."

DO NOT add `loading.tsx` or `error.tsx` files for this route — the route group already has them shipped from Phase 1. Verify they exist (check `src/app/(protected)/loading.tsx` and `error.tsx`) but don't re-create.

3. **`src/app/login/actions.ts`** — `'use server'`. Función `loginAction(prev, formData)`:
   - **Paso 1**: Leer IP del request via `headers()` — `x-forwarded-for` (split por coma, primera) o fallback a `'unknown'`.
   - **Paso 2**: `const limit = await loginLimiter.limit(ip);` — Si `!limit.success`, return `{ error: 'Demasiados intentos. Intenta de nuevo en unos minutos.' }`.
   - **Paso 3**: Validar formData con Zod (`z.object({ password: z.string().min(1) })`). Si falla, return `{ error: 'Password requerido' }`.
   - **Paso 4 (timing-safe)**: SIEMPRE ejecutar `bcrypt.compare(input, process.env.DASHBOARD_PASSWORD_HASH ?? '$2a$10$invalidhashinvalidhashinvalidhashinvalidhashinvalidhash')`. NO short-circuitear si el env var falta — corre bcrypt con un hash dummy para que el tiempo de respuesta sea constante.
   - **Paso 5**: Si `compare` true: `const token = await encrypt({ authed: true }); (await cookies()).set('session', token, getSessionCookieOptions()); redirect('/');`.
   - **Paso 6**: Si false: return `{ error: 'Password incorrecto.' }`.
   - **Importante**: `redirect()` lanza una excepción especial — debe estar fuera de try/catch o usar `redirect()` después del cookie set sin atraparla.

4. **`src/app/logout/route.ts`** — Route Handler GET y POST que llama `(await cookies()).delete('session')` y `redirect('/login')`. Acepta GET para que un `<a href="/logout">` simple funcione.

5. **`proxy.ts` en raíz del proyecto** (NO en src/, NO middleware.ts):
   ```ts
   import { NextRequest, NextResponse } from 'next/server';
   import { decrypt, SESSION_COOKIE_NAME } from '@/lib/auth/session';

   const PUBLIC_PATHS = ['/login'];

   export default async function proxy(req: NextRequest) {
     const { pathname } = req.nextUrl;
     if (PUBLIC_PATHS.some(p => pathname === p || pathname.startsWith(p + '/'))) {
       return NextResponse.next();
     }
     const cookie = req.cookies.get(SESSION_COOKIE_NAME)?.value;
     const payload = await decrypt(cookie);
     if (!payload?.authed) {
       const url = req.nextUrl.clone();
       url.pathname = '/login';
       return NextResponse.redirect(url);
     }
     return NextResponse.next();
   }

   export const config = {
     matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
   };
   ```

**Anti-pattern a evitar**: NO setear cookie dentro de proxy.ts (Next.js docs: el flow de login DEBE ir por Server Action o Route Handler, nunca proxy).

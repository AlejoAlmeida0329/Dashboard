**Component 1 — `src/components/bonos/KPICardsV2.tsx` (Server Component):**

Renders 5 KPI cards in a responsive grid (1 col mobile → 2 col sm → 5 col xl). Order:
1. **Bonos recibidos** — count + monto subtitle. Accent: `text-section-bonos`. Always visible.
2. **Bonos enviados** — count + monto subtitle. Accent: `text-section-bonos`. Always visible.
3. **Volumen recibido (COP)** — formatCOP(montoIn). Accent: `text-section-bonos`. Always visible.
4. **Volumen enviado (COP)** — formatCOP(montoOut). Accent: `text-section-bonos`. Always visible.
5. **Ticket promedio** — formatCOP(ticketPromedio). Accent: `text-muted-foreground`. Always visible.

Empty-state copy at the bottom of each card if `(countIn + countOut) === 0`: "Sin bonos en el período" — keep the card visible (zero is still data) so the layout is stable across filter changes.

Props: `{ summary: BonoSummaryV2 }` imported from `@/lib/domain/bonos`. Export named `KPICardsV2`.

**Component 2 — `src/components/bonos/TopEmisores.tsx` (Server Component):**

Renders a single Card titled "Top emisores" with a Table inside. Columns: `#` (rank, 1-indexed), `Tikintag`, `Bonos` (count, right-aligned), `Volumen` (formatCOP, right-aligned). Row count: as many as supplied (caller decides — typical n=10).

Visual treatment per CONTEXT.md "protagonista":
- Card with `border-l-4 border-section-bonos` (left accent stripe in section color)
- Generous header padding; table tight (`text-sm`).
- Rank column: small monospace number badge (`tabular-nums` + `text-muted-foreground`).
- Tikintag column: prominent (`font-medium`).

Empty-state row inside the table when `rows.length === 0`: a single TR spanning all 4 cells with copy "Sin emisores en el período" centered.

Props: `{ rows: BonoTikintagRow[] }`. Export named `TopEmisores`.

**Component 3 — `src/components/bonos/TopReceptores.tsx` (Server Component):**

Identical structure to `TopEmisores`. Title "Top receptores". Empty-state copy "Sin receptores en el período". Same `border-l-4 border-section-bonos` accent. Export named `TopReceptores`.

You may extract the shared body into a private helper inside one of the two files OR into a new `_BonoRankingTable.tsx` if it keeps both components <40 LOC each. Either path is fine; do not over-abstract.

**Component 4 — `src/components/bonos/BonosFlowChart.tsx` (Client Component — `"use client"`):**

Stacked-bar timeline of `enviados` vs `recibidos` per day using Recharts `BarChart` (the same library `BonosChart.tsx` already uses — `recharts` is in `package.json`). Two stacked series:
- `countOut` colored with `var(--section-bonos)` at OKLCH lightness 0.50 (the "enviados" stack, using the section base color)
- `countIn` colored with `var(--section-bonos)` at lightness 0.65 (the "recibidos" stack, lighter shade)

You can grab both shades by reading `--section-bonos` once for the darker stack and applying a Tailwind `opacity-70` to the lighter one OR by hard-coding two OKLCH values (preferred for chart fidelity — Recharts doesn't compose Tailwind opacity gracefully on bars). Same pattern: define two constants at the top of the file and pass them as `fill` props.

X-axis: ISO date `YYYY-MM-DD` from `BonoByDateV2.date` (Bogotá day key). Y-axis: count. Tooltip: shows date + countIn + countOut + monto totals (formatCOP-formatted).

Props: `{ data: BonoByDateV2[] }`. Export named `BonosFlowChart`.

When `data.length === 0`: render a simple `<p>` with copy "Sin actividad en el período" instead of an empty chart frame (avoid Recharts' unhelpful empty-axis appearance).

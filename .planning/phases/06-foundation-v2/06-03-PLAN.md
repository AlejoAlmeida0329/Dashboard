  <name>Task 1: Extend DashboardFilters and URL helpers in url-state.ts</name>
  <files>src/lib/url-state.ts</files>
  <action>
Extend the URL state contract with two new optional array fields. Keep ALL existing behavior intact (from, to, empresa, presenter must continue working unchanged).

1. **Update `DashboardFilters` type:**
   ```ts
   export type DashboardFilters = {
     from?: string;
     to?: string;
     empresa?: string;
     status?: string[];   // NEW — e.g. ['completed', 'failed']
     tipo?: string[];     // NEW — e.g. ['BONUS', 'P2P']
     presenter?: "1";
   };
   ```

2. **Update `parseFilters()`:**
   - Add CSV parsing helper:
     ```ts
     const getCSV = (sp: Record<string, string | string[] | undefined>, k: string): string[] | undefined => {
       const raw = getOne(sp, k);
       if (!raw) return undefined;
       const parts = raw.split(",").map(s => s.trim()).filter(s => s.length > 0);
       return parts.length > 0 ? parts : undefined;
     };
     ```
   - Read `status` and `tipo` via `getCSV`. Empty CSV → `undefined` (treat absent and empty-set the same — "no filter applied").

3. **Update `buildUrl()`:**
   - After `empresa`, before `presenter`, append:
     ```ts
     if (filters.status && filters.status.length > 0) params.set("status", filters.status.join(","));
     if (filters.tipo && filters.tipo.length > 0) params.set("tipo", filters.tipo.join(","));
     ```
   - This preserves stable URL ordering: from → to → empresa → status → tipo → presenter.

4. **Update JSDoc** at the top of `url-state.ts` to mention the two new params and the CSV serialization convention.

5. **Do NOT touch** `presetDateRange`, `detectActivePreset`, or `BOGOTA_TZ` — those are date-range concerns and out of scope.
  </action>

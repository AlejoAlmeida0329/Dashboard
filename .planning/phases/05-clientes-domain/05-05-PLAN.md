    <option id="subdomain-tikin">

  <action>
    Run resolution + HTTPS smoke (after the user types "done" in Task 3):

    ```bash
    # 1. DNS resolution
    dig <DOMAIN> +short
    # Expect: cname.vercel-dns.com (CNAME) or 76.76.21.21 (A)

    # 2. Vercel domain status: should be "Verified" (or auto-flips after DNS propagates)
    vercel domains ls

    # 3. HTTPS smoke (Vercel auto-provisions SSL once DNS verifies; usually <2 min)
    curl -sI https://<DOMAIN>/login | head -3
    # Expect: HTTP/2 200 ... (or HTTP/1.1 200 with no SSL warning)

    # 4. Proxy gate intact (unauthenticated /clientes redirects to /login)
    curl -sI https://<DOMAIN>/clientes | head -3
    # Expect: HTTP/2 307 ... Location: /login
    ```

    If the Vercel `vercel domains ls` shows "Pending" past 5 min after `dig` shows correct propagation, run `vercel certs issue <DOMAIN>` to nudge SSL (rare; Vercel usually auto-issues).

    If `curl` returns SSL errors or `HTTP/2 526` (Cloudflare-style cert error), wait 2 min for SSL provisioning and retry.
  </action>

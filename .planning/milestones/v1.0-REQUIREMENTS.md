# Requirements: Tikin Dashboard

**Defined:** 2026-04-27
**Core Value:** Una sola URL donde el equipo de Tikin ve métricas frescas del negocio (bonos vendidos, recargas, payouts, clientes activos) sin tener que abrir Sheets ni armar reportes ad-hoc — y que se vea presentable cuando se proyecta a clientes.

## v1 Requirements

### Cross-cutting

- [x] **CROSS-01**: Filtro global de rango de fechas (7d / 30d / MTD / QTD / YTD / custom) que aplica a todas las pestañas
- [x] **CROSS-02**: Filtro global de empresa (single-select, sticky entre pestañas, codificado en URL para compartir)
- [x] **CROSS-03**: Formato consistente de moneda (COP, separador de miles, sin decimales en números grandes) y fechas en es-CO
- [x] **CROSS-04**: Timestamp visible de "última lectura" en cada vista (data en vivo desde Sheets en cada page load)
- [x] **CROSS-05**: Estados de loading, empty y error decentes en cada widget (no spinners crudos ni pantalla en blanco)
- [x] **CROSS-06**: Toggle "Modo Presentación" que oculta widgets internos (revenue, comisión, take rate, leaderboards de otras empresas), agranda fuentes y limpia chrome de navegación

### Authentication

- [x] **AUTH-01**: Password compartido único protege el acceso a todo el dashboard
- [x] **AUTH-02**: Sesión persiste en cookie HttpOnly (no requiere re-login en cada page load)
- [x] **AUTH-03**: Login tiene rate limiting (5 intentos / 5 min / IP) — implementado con Upstash sliding-window

### Infrastructure

- [x] **INFRA-01**: Despliegue en Vercel (etapa de prueba) — last production: https://project-dashboard-z0fpsm5hl.vercel.app
- [x] **INFRA-02**: Conexión a las dos hojas de Google Sheets (BD_Plataforma + BD_Payouts) vía `googleapis` `batchGet`
- [x] **INFRA-03**: Validación de schema en cada lectura de Sheets — Zod by header name (no índice); falla claro si una columna cambia
- [ ] **INFRA-04**: Migración a dominio propio (ej. `dashboard.tikin.co`) — **DEFERRED**: Plan 05-05 pospuesto por decisión del usuario; cliente-flow LIVE en `.vercel.app`. Pendiente decisión de qué dominio usar.

## Out of Scope

| Feature | Razón |
|---------|-------|
| Login por usuario (email/password individuales o OAuth) | Password compartido es suficiente para uso interno + presentaciones |
| Acceso 24/7 de clientes finales | No es portal de cliente — solo se proyecta en llamadas |
| Multi-tenant con cuentas / vistas propias por cliente | Se resuelve filtrando data, no creando cuentas |
| Escritura / edición de data desde el dashboard | Solo lectura — Google Sheets sigue siendo la fuente de verdad |
| Integración directa con el sistema core de Tikin | Sheets como fuente intermedia |
| Cache / refresh diferido | Lectura en vivo en cada page load — volumen de tráfico bajo justifica latencia |
| Versión móvil dedicada | Responsive básico es suficiente |
| Real-time / WebSocket updates | Polling es overkill para uso interno |
| Drill-down a empleados / usuarios finales | PII — alto riesgo en pantalla compartida con cliente |
| ML forecasting / anomaly detection | Riesgo de alucinación con cliente; data no suficientemente limpia |
| Botón de retry / acción manual sobre payouts fallidos | Solo lectura |
| Dashboards configurables / drag-drop por usuario | Las 5 pestañas SON los dashboards |
| Alertas / notificaciones in-app | No es producto de monitoreo |
| Drill-down a lista cruda de transacciones | Riesgo de PII |
| Per-user role-based widget visibility | Modo Presentación cubre el caso |
| Goal/target lines en gráficas | Targets requieren governance que no se va a mantener |
